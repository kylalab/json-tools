중첩 JSON 평탄화(Flatten) 전략

2025-10-03 · 작성자 Kyla

JSON은 중첩 구조가 자유롭지만, 분석팀이나 시트/데이터베이스에서는 이를 표(테이블) 형태로 다뤄야 합니다. 이때 필요한 것이 JSON 평탄화(Flatten) 전략입니다.

평탄화 기본 규칙

  • 경로 키 생성: 중첩 객체는 `.` 또는 `_` 로 이어붙임 (a.b.c)
  • 배열 처리: 인덱스 번호로 구분 (items[0].price)
  • null/빈 배열: 일관되게 공백 혹은 `null` 문자열 처리

실제 예시

원본 JSON

{
  "id": 1,
  "user": {
    "name": "Alice",
    "contacts": [
      {"type": "email", "value": "alice@test.com"},
      {"type": "phone", "value": "010-1234-5678"}
    ]
  }
}

평탄화된 결과

id,user.name,user.contacts[0].type,user.contacts[0].value,user.contacts[1].type,user.contacts[1].value
1,Alice,email,alice@test.com,phone,010-1234-5678

충돌 방지

  • 점 포함 키: 원래 키에 점이 있으면 따옴표 처리 또는 다른 구분자 사용
  • 배열 vs 객체 혼합: 동일 경로에 배열과 객체가 섞이지 않도록 설계
  • 유니코드/특수문자: 키 이름 표준화(소문자, `_` 권장)

실무 팁

  • 분석팀: 열 이름을 문서화해두고, 변환 로직을 공유
  • 개발팀: API 응답 스키마에 배열 처리 규칙을 명시
  • QA팀: 표본 데이터를 CSV로 내려서 엑셀/시트에서 검증

Quick JSON의 도구를 이용하면 브라우저 내에서 안전하게 JSON을 CSV로 변환하고, 위와 같은 평탄화 결과를 바로 확인할 수 있습니다.
