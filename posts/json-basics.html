JSON 기본 개념과 자주 하는 실수 8가지

2025-09-28 · 작성자 Kyla

JSON(JavaScript Object Notation)은 가볍고 사람이 읽기 쉬운 데이터 형식입니다. API 응답, 모바일·웹 앱, 서버 간 통신에서 표준처럼 쓰입니다.

JSON 핵심 문법 요약

  • 객체는 { }, 배열은 [ ]로 감싼다.
  • 문자열은 "더블쿼트"만 허용한다.
  • 마지막 원소 뒤 쉼표 금지 (Trailing comma X)
  • 한 줄 주석, 끝에 세미콜론 등은 허용되지 않는다.

실무에서 자주 하는 실수 8가지

  1. 키/문자열에 단일 따옴표(') 사용
  2. 마지막 원소 뒤에 쉼표 붙이기
  3. undefined, NaN 같은 JS 전용 값 사용
  4. 날짜를 문자열로 감싸지 않음(표준 JSON엔 Date 리터럴 없음)
  5. 중첩 구조 괄호 불균형
  6. 배열과 객체를 혼합해 일관성 없는 스키마
  7. 큰 숫자 정밀도 이슈(긴 ID는 문자열로 다루기)
  8. 서버/클라이언트 인코딩 불일치(UTF-8 권장)

유효성 검사를 생활화하세요
